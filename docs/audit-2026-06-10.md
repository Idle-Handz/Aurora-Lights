# Audit follow-up - 2026-06-10

Updated 2026-06-15 after re-checking the original audit against the current
Aurora Reflections codebase.

## Remaining relevant items

### Source release metadata still uses textual MAX

`Aurora.App/Services/DbElementLoader.cs` still computes picker tiebreak metadata
with `MAX(se.release_text)`. This is fine for ISO-like strings and single-release
sources, but non-ISO release labels can sort alphabetically instead of
chronologically. The impact is limited to ordering duplicate display names.

## Fixed or no longer relevant

- Session notes persistence is fixed by blur-save.
- Malformed `.index` files no longer abort the whole content check.
- Content cache metadata write failures no longer make a successful download
  look failed.
- `source_file_id` is now included in the database schema guard.
- Temp HP changes now re-clamp current HP to the merged HP pool.
- Spell-slot usage now clamps to the current slot total before display/toggle,
  preventing negative remaining-slot counts after level or feature changes.
- ASI cleanup no longer swallows unregister/selection-slot failures silently, and
  no longer reports an invalidated choice when unregistering did not succeed.
- Attack reminders now dedupe generated weapon rows by full reminder text:
  identical rows collapse, but same-weapon rows with different name, attack,
  damage, or range remain selectable. Legacy identifier-only selections still
  select all generated rows for that weapon.
- `UpgradeIndexFileProtocols` now rewrites only `url=` attribute values instead
  of every `http://` occurrence in an `.index` file.
- Content index downloads now apply a per-entry body timeout in
  `Aurora.Logic`, so a stalled file is reported as failed while later entries
  can continue.
